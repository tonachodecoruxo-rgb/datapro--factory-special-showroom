## 1. CONCEPTO / PROBLEMA
Los agentes de IA con acceso a herramientas (APIs, bases de datos) son vulnerables a ataques de inyección de instrucciones. El problema radica en la dificultad de distinguir entre una instrucción legítima y un vector de ataque que busca el abuso de herramientas (Tool Abuse) o el bypass de autenticación.

## 2. DOSSIER TÉCNICO
- **Volumen:** 5.000 registros de trazas de ataque.
- **Vectores de Ataque:** Prompt Injection, Tool Abuse, Auth Bypass, Context Poisoning.
- **Metadatos Clave:** Razonamiento interno del agente (`thought`), estado de infiltración y flags de prevención.
- **Esquema:** JSON jerárquico compatible con RSR (Real-time Sensor Reliability).

## 3. DOSSIER ROI (5 AÑOS)
La implementación de este dataset para el entrenamiento de firewalls agénticos reduce el riesgo de brechas de seguridad en un 40%, ahorrando una media de 1.2M€ en costes de respuesta ante incidentes y cumplimiento legal.
